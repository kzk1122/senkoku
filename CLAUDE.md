# CLAUDE.md − senkoku(線刻)開発ガイド

## プロジェクト概要

**線刻(せんこく / senkoku)** は、Apple Pencil でイラストの基礎を段階的に練習する Web アプリ(PWA)。
書道をモチーフにした「道場」形式で、課題をクリアすると次が解放される。

- 開発者はイラストほぼ初心者。自分自身の練習用に開発している
- リポジトリ: https://github.com/kzk1122/senkoku
- 公開先: Cloudflare Pages(静的ホスティング、ビルド設定なし、output directory `/`)

## 全体構想(3ステージ)

1. **Stage 1: 運筆道場**(実装済みプロトタイプ)− 線をなぞって採点。線画・運筆の基礎
2. **Stage 2: 形体道場**(未実装)− 図形分解による人体アタリ、タイマー式ジェスチャードローイング(30秒〜2分)
3. **Stage 3: 模写道場**(未実装)− お手本を「隣に表示 → 薄く下敷き → 記憶模写」と段階的に難化。完成後に差分オーバーレイ
4. 将来構想: Claude API による描画講評機能

## 技術方針(重要・変更しないこと)

- **Vanilla JS / ビルドツールなし**の静的サイト。npm 依存を導入しない
- 開発者の既存プロジェクト(Iemono, HibiOri)と同じ系統: 素の Web 技術 + Cloudflare Pages
- UI は日本語。デザインは「和紙 × 墨 × 朱」(css/style.css の CSS 変数がデザイントークン)
  - `--paper: #F6F1E4` / `--ink: #221F1A` / `--shu: #C0392F`(朱=添削印の色)/ `--nibi: #8B8577`
  - フォント: Shippori Mincho(見出し)+ Zen Kaku Gothic New(本文)
- 採点結果は「朱印スタンプ」(秀・優・良・可・再)で表示するのがアプリの象徴的 UI
- データ保存は現状 localStorage(`senkoku_progress_v1`)。**スコア履歴・練習日数を持つ段階で IndexedDB へ移行予定**

## Stage 1 の実装仕様(js/app.js)

### 課題

`COURSES` 配列で定義。各課題は `pathFn(t)` (t∈[0,1] → 正規化座標 0〜1) でお手本パスを生成。
現在6課題: 横画 → 縦画 → 払い → 波線 → 円相 → 渦巻。**70点以上で次の課題が解放**(PASS_SCORE)。

### Apple Pencil 入力

- PointerEvent 使用。`pressure` で線の太さを変える: `lineWidth = 2 + 7 * pressure`(マウスは 0.5 相当)
- `getCoalescedEvents()` で高解像度サンプリング
- パームリジェクション: 一度 `pointerType === "pen"` を検知したら以降 touch を無視(state.penSeen)
- canvas は `touch-action: none`、devicePixelRatio でスケーリング
- 一筆書き方式: pointerdown で前の線をクリア、pointerup で自動採点(8点未満の線は無視)

### 採点アルゴリズム(scoreStroke)

お手本パス 240 点サンプルとの比較。距離はお手本のバウンディングボックス対角線 `size` で正規化。

| 指標 | 重み | 計算 |
|---|---|---|
| 精度 acc | 50% | 描画各点→お手本の平均最短距離。`clamp01(1 - (meanErr - 0.008) / 0.05) * 100` |
| 網羅 cov | 30% | お手本各点の 5.5% 近傍を通過した割合 |
| 滑らか smo | 20% | 前平滑化(step/2 で等間隔リサンプリング → 3点移動平均×2)した線の角度二階差分(step = size/45)から、お手本自身の曲率を引いた余分なガタつき。`clamp01(1 - excess / 0.30) * 100` |

**網羅ゲート**: `total = round((acc*0.5 + cov*0.3 + smo*0.2) * clamp01((cov - 40) / 50))`
→ 線を途中でやめると大幅減点。「一筆で描き切る」ことを最優先で評価する思想。

検証済みの挙動(この値を壊さないこと): 完璧トレース=100点 / 軽い人間的ブレ・丁寧なゆっくり線=90点台(smo 70〜99)/ 雑な線=smo 0〜20 / 半分で放棄=20点台。
※滑らかさは 2026-07 に実機調整済み: 前平滑化なしだと Apple Pencil のセンサノイズ+手ブレで丁寧な線でも smo=0 に張り付いた。挙動変更時は Node シミュレーション(合成ストロークで上記シナリオを再現)で回帰確認すること。

ランク: 90+ 秀 / 80+ 優 / 70+ 良(合格)/ 55+ 可 / 未満 再。

### その他の UI

- 補助線3段階トグル(濃 0.5 / 淡 0.18 / 無 0)。始点●と方向矢印は常時表示

- 課題レール(chip)に最高得点と解放状態を表示

## 開発時の注意

- ローカル確認: `python3 -m http.server 8080`(8000 は他の node プロセスと衝突しがち)。iPad からは `http://<MacのIP>:8080`
- Service Worker のキャッシュが効くので、動作確認時はハードリロード or CACHE 名の更新を忘れない
- **`hidden` 属性は author CSS の `display` 指定に負ける**。オーバーレイ等の表示切替に `hidden` を使う場合は css/style.css の `[hidden] { display: none !important; }` が前提(過去にこれが原因で、非表示のはずの結果オーバーレイが canvas を覆いペン入力を奪うバグがあった)
- コミットメッセージ・コード内コメントは日本語で OK
